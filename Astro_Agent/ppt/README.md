# PPT 生成指南 · Astro Agent 工具箱 + 个人简历

> 给执行人（PPT 生成 / 设计同事）的完整说明书。本目录里的所有 `slides_prompts/*.md` 都是**单页一图**的「图像生成 Prompt」。一张图 = 一张 PPT 页面，按顺序拼起来即是整套汇报 PPT。
>
> 不要再用 `slides`/`pptx` 类自动排版工具；目标是让最新一代图文生成模型（GPT-Image / Imagen / 即梦 / Seedream / Nano-Banana 等）直接产出**整页 16:9 PNG**，导入 PowerPoint 占满整页即可。

---

## 0. 项目背景（一句话给生成人看）

我（林佳茂，中山大学物理与天文学院 2024 级博士）正在为老板汇报一套**面向白矮星 / 双星 / 多波段巡天**的天文科研全栈平台 `Astro_Agent`。它由三层组成：

1. **astro_toolbox** — 30+ 模块的 Python 工具箱，统一封装 SDSS/DESI/LAMOST/KOA/HST/JWST/ZTF/TESS/Gaia/WISE/X-ray 等查询、SED/HR/WD 拟合、周期 / RV / 轨道回溯。
2. **RAG 数据库 + 知识图谱（KG）** — 本地白矮星文献 SQLite 库 + `prompt2graph_for_astronomy` 多阶段去重知识图谱，用来给 Agent 注入领域方法论。
3. **analysis_agent (Chief Investigator)** — 基于 LangGraph 的多节点工作流，融合 PaperOrchestra 五智能体写作管线 + Codex 风格安全门，可以从「目标坐标」一路跑到「ApJ 风格论文草稿 + 同行评审意见」。

汇报对象是公司 / 实验室老板，目标是让他在 15–20 分钟内理解：**我在做什么、技术深度多深、跟我之前做的天文大模型 (maoAstro LLM) 是怎么衔接的**。

## 2. 全局视觉规范（**两套 PPT 完全一致，不要破例**）

| 项目 | 取值 |
| --- | --- |
| 比例 | **16:9**（建议 1920×1080 或 2560×1440） |
| 背景 | **纯白 #FFFFFF**，不要渐变、不要纸纹 |
| 主题色 | **薄荷色 RGB(18, 204, 185) ≈ #12CCB9**；用作主标题、关键数字、icon、强调线、箭头 |
| 辅助色 | 深灰 #2F3A45（正文）、浅灰 #ECEFF3（卡片底 / 分割线）、强调红 #FF6B6B（仅用于「痛点 / 缺陷 / Before」对比） |
| 字体 | 中文 **思源黑体 / Source Han Sans**；英文 **Inter / Helvetica Neue**；标题粗体，正文常规 |
| 主标题 | 左上角，36–44pt，**前面带一条 8px × 64px 的薄荷色短竖条**作为视觉锚点 |
| 英文副标题 | 主标题下方一行，浅灰 18pt，斜体，例：`End-to-End Architecture` |
| 页脚 | 右下角小字 `林佳茂 · 中山大学物理与天文学院 · 2026` + 极细薄荷色横线 |
| **排版灵魂** | **绝对禁止**「左右两栏 / 三栏并列」死板布局！每一页都用**非对称**：偏轴卡片 + 流程箭头 + 圆环 + 分支树 + 叠加圆 + 时间轴 + 同心圆 + Z 字流，**至少 3 种构图**在整套 PPT 里轮换 |
| 信息密度 | 每页 ≤ 5 个文字块，每块 ≤ 25 个汉字，**不允许大段文字**；要点全部图形化 |
| Icon 风格 | 线性 icon（线宽 2px，圆角），统一薄荷色描边、白底；不要彩色实心 icon、不要 emoji |
| 数字强化 | 关键指标用「特大薄荷色数字 + 小灰色单位 + 一行解释」方式呈现 |

## 3. 命名 & 工作流

执行人请按以下顺序操作：

1. 打开 `slides_prompts/A_toolbox_deck/01_cover.md`，复制其中的 **Prompt 段** 给图像模型；
2. 检查 `参考图` 字段，如果列出了 `reference_images/xxx.png`，把那张图作为 **图像参考 / inpaint 底**一并喂给模型；
3. 模型出图后保存为同名 PNG（`01_cover.png`）放到 `slides_prompts/A_toolbox_deck/output/` 下；
4. 全部出完再统一塞进 PowerPoint：每页一张图，**满版铺底，0 边距**，不要再叠任何 PPT 文本框。

如果某一页出图与文字不符（图文模型偶尔会写错关键词），请按 `slides_prompts/xxxx.md` 里给出的「关键中文文字（必须出现）」字段反复重生成，直到关键术语正确为止。

## 5. 给执行人的若干硬性约束

- **薄荷色绝对值**：(R=18, G=204, B=185)，转十六进制 `#12CCB9`；如果模型出图偏蓝绿，请在 prompt 里加「mint green RGB 18 204 185, NOT teal, NOT cyan」。
- **不要英文单词错拼**：所有英文术语见各 prompt 文件的「英文术语锁定清单」。
- **不要出现真人脸**（除了 B 套封面可使用 `reference_images/image1.jpeg` 我的证件照作参考）。
- **数字一字不改**：CV 里写的 89%、6%、3.95/5.0、87.44、28 万、6.2 万、12G 显卡等统统照搬。
- **不要加入虚构 Logo**；可以用「中山大学物理与天文学院」中文校名作为页眉小字。
- 每张图右下角统一打 `林佳茂 · 2026`，无别的水印。
