==================================================

Always follow this order:

Step 2. Identify one or more true MaterialSystem nodes for each SourceDocument.

Step 5. Attach sample-specific scalar facts to the merged ExperimentalSample nodes.
Step 6. Attach shared family-level non-scalar findings to the MaterialSystem.
Step 7. Attach characterization / structure / property / mechanism / limitation nodes around the sample ring.
Step 8. Add cross-paper concept reuse only through shared concept-level nodes, not by merging unrelated samples.

Never skip identity resolution.

==================================================

This is a unified graph across many papers, not separate graphs.

==================================================
3. MATERIAL SYSTEM RULE
==================================================

A MaterialSystem is the central experimental platform or sample family within one SourceDocument.

Use MaterialSystem for:
- FeCu/GDY family
- FeCoNiCuZn/GDY family
- Fe3C microflake family
- SA/graphene family
- HCNF/HEA family

A paper may contain more than one MaterialSystem if and only if the paper truly studies multiple distinct central systems.

Do NOT create MaterialSystem nodes for:
- background-only literature examples
- generic review phrases
- author names
- vague words like “materials”, “composites”, “supports”

Every ExperimentalSample must derive_from exactly one MaterialSystem.

==================================================
4. SAMPLE-RING RULE
==================================================

ExperimentalSample nodes must surround a MaterialSystem.

Correct backbone:
SourceDocument -> MaterialSystem -> ExperimentalSample

Then:
ExperimentalSample -> CharacterizationMethod / CharacterizationFinding / StructureConcept / EMProperty / LossMechanism / Limitation

Do NOT create:
- free-floating ExperimentalSample nodes
- umbrella samples carrying an entire series
- MaterialSystem nodes that absorb sample-level measured scalar values when explicit samples exist

==================================================
5. MANDATORY SAMPLE MERGING / COREFERENCE RULE
==================================================

This is the most important rule.

If multiple mentions in the text refer to the same experimental sample, you MUST merge them into one canonical ExperimentalSample node.

Merge all of the following when they refer to the same sample:
- full name vs short name
- code name vs descriptive phrase
- “the sample”
- “this sample”
- “it”
- “the 800°C sample”
- “the 50 wt% loading sample”
- “the optimal sample”
- “the Cu-only/GDY sample”
- “the counterpart”
- repeated mentions across neighboring sentences
- formatting/OCR variants
- family label + unique condition label

Examples:
- “sample annealed at 800°C for 4 h”
- “the 800°C sample”
- “this sample”
-> one canonical sample

- “50 wt% loading sample”
- “the optimal loading sample”
-> one canonical sample, e.g. Sample_FeCu_GDY_L50

- “Cu-only/GDY sample”
- “the Cu counterpart”
-> one canonical sample if context shows same object

Use:
- sample_name = one canonical name only
- sample_aliases = all other textual variants
- coreference_note = brief note when the merge depended on contextual reasoning

Never create duplicate sample nodes that differ only by:
- wording
- formatting
- abbreviation
- pronoun reference
- minor OCR variation
- descriptive alias

9. FIELD LEGALITY RULE
==================================================

Use scalar fields conservatively and legally.

Allowed:
- RL_min_value -> one numeric dB value only
- RL_min_frequency -> one numeric GHz point only
- EAB_value -> one numeric GHz width only
- EAB_frequency_range -> one GHz interval only
- matching_thickness -> one absorber thickness only
- particle_size -> explicit size only
- filler_loading -> wt% loading only
- metal_molar_ratio -> true composition ratio only
- precursor_loading_ratio -> true ratio only
- precursor_amount_note -> raw amounts such as “1.5 mmol CuCl, 500 mg TEB”

Forbidden:
- descriptive phrases in numeric fields
- “much narrower” in EAB_value
- “significantly smaller” in particle_size
- “specified” in filler_loading
- “500 mg TEB” in filler_loading
- “below 70%” in RL_min_value when it is actually absorption_rate
- “peak at 11.6 GHz” in epsilon_imag if it is a peak position rather than a value
- “outer diameter / inner diameter” in matching_thickness

If a statement is descriptive rather than a legal scalar value, store it in:
- performance_note
- property_note
- remarks
