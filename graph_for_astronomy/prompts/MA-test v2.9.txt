Characterization extraction rule:
For each characterization method, extract not only the method itself, but also the main finding or conclusion explicitly supported by that method whenever stated in the text.

This includes:
- what the method measures
- what structural / compositional / mechanistic conclusion it supports
- which sample the characterization result belongs to

Examples:
- XRD -> supports phase identification, peak weakening/disappearance, alloying, interdiffusion, dominant phase conclusions
- ICP -> supports elemental ratio conclusions, trace-element conclusions, non-equiatomic composition conclusions
- Cole-Cole analysis -> supports dielectric relaxation conclusions
- Electron microscopy -> supports particle size, aggregation, dense morphology, dispersion conclusions

Naming rules:
1. Use fully unified prefixes. Do not mix alternative prefixes.

2. MaterialSystem names must always use this exact format:
   "System_{composition}_{support}"
   Example:
   "System_FeCu_GDY"

3. ExperimentalSample names must always use this exact format family:
   "Sample_{composition}_{support}_{main_variable}"
   Never use prefixes such as:
   - ExperimentalSample_
   - MaterialSystem_
   - ExperimentalSample_Sample_

4. ExperimentalSample naming must follow compressed naming logic.
   Do NOT include all conditions in the sample name by default.

5. Identify the main comparison variable of the experimental series:
   - If the study section mainly compares annealing temperature, use temperature in the sample name.
   - If the study section mainly compares annealing time, use time in the sample name.
   - If the study section mainly compares filler loading, use loading in the sample name.
   - If two variables are explicitly varied together and jointly define the comparison, include both.
   - Keep other fixed background conditions as attributes, not as part of the name.

6. Preferred ExperimentalSample naming formats:
   - Temperature series:
     "Sample_{composition}_{support}_T{temperature}"
     Example: "Sample_FeCu_GDY_T600"

   - Time series:
     "Sample_{composition}_{support}_t{time}"
     Example: "Sample_FeCoNiCuZn_GDY_t4h"

   - Loading series:
     "Sample_{composition}_{support}_L{loading}"
     Example: "Sample_FeCu_GDY_L50"

   - Joint-variable series:
     "Sample_{composition}_{support}_T{temperature}_t{time}"
     Example: "Sample_FeCoNiCuZn_GDY_T800_t4h"

7. Use one and only one naming logic within the same experimental series.
   Do not mix naming styles such as:
   - Sample_FeCu_GDY_600C
   - Sample_FeCu_GDY_600C_50wt
   within the same temperature-comparison series if loading is fixed.

8. If a condition is constant across the compared series, store it as an attribute rather than in the sample name.
   Examples:
   - If filler loading is fixed across a temperature series, do not include loading in the sample name.
   - If annealing time is fixed across a temperature series, do not include time in the sample name.

9. Use the original composition order as stated in the text. Do not permute element order.
