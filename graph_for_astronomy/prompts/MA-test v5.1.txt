==================================================
2. WHAT IS A MATERIALSYSTEM VS AN EXPERIMENTALSAMPLE
==================================================

MaterialSystem:
Use for the general material platform or family that organizes multiple related samples in the paper.

Typical signs of MaterialSystem:
- it refers to a family, series, platform, or general composite system
- multiple samples are derived from it
- it is used to organize the results section
- it is not just one single tested instance

Examples:
- FeCu/GDY family
- FeCoNiCuZn/GDY composites
- Co SA/graphene family
- HCNF/HEA family
- Fe3C microflake family

ExperimentalSample:
Use for one concrete tested sample.

Typical signs of ExperimentalSample:
- it has its own condition, code, temperature, time, ratio, loading, or composition
- it has its own measured performance
- it is a control or comparison sample
- it has sample-specific characterization results
- it is discussed as one actual tested object

Examples:
- sample annealed at 800°C for 4 h
- 50 wt% loading sample
- Cu-only/GDY sample annealed at 1000°C
- pristine graphene used as comparison sample
- FC-700

Important:
- A family/platform is not a sample.
- A tested comparison sample is still an ExperimentalSample, even if its composition is simple.
- A precursor can be an ExperimentalSample if the paper explicitly treats it as one experimental sample object.
- A support material such as graphene, GDY, or HCNF is usually CarbonSupport unless the paper explicitly tests it as a standalone sample.

3. HARD SAMPLE MERGING RULE
==================================================

If multiple mentions refer to the same experimental sample, you MUST merge them into one canonical ExperimentalSample node.

Merge all co-referential mentions, including:
- full name vs short name
- sample code vs descriptive phrase
- “the sample”
- “this sample”
- “it”
- “the 800°C sample”
- “the 50 wt% loading sample”
- “the optimal sample”
- “the counterpart”
- nearby repeated mentions
- minor formatting or OCR variants

If they refer to the same real sample, they must become one node.

Never create duplicate samples that differ only by:
- wording
- abbreviation
- pronoun reference
- descriptive alias
- minor OCR variation

==================================================

Use scalar fields conservatively and legally.

- “500 mg TEB” -> not filler_loading

- “3.0 mmol Fe and 1.5 mmol Cu” -> not metal_molar_ratio unless explicitly given as a true ratio

If the statement is descriptive rather than a legal scalar value, place it in:
- performance_note
- property_note
- remarks
