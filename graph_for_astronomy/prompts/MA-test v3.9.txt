==================================================
A. EXTRACTION ORDER
==================================================

Always follow this order:

Step 1. Identify SourceDocument boundaries.
Step 2. Identify one or more MaterialSystem nodes for each SourceDocument.
Step 3. Identify all ExperimentalSample mentions belonging to each MaterialSystem.
Step 4. Resolve sample aliases, shorthand names, descriptive mentions, and pronouns into canonical ExperimentalSample names.
Step 5. Attach sample-specific scalar facts to the merged ExperimentalSample nodes.
Step 6. Attach shared family-level non-scalar findings to the MaterialSystem.
Step 7. Attach characterization, structure, property, mechanism, and limitation nodes around the sample ring.

Never skip identity resolution.
Never extract sample facts first and merge later.
First resolve identity, then write triples and attributes.

==================================================

==================================================

If the chunk contains multiple papers, multiple abstracts, separate titles, explicit labels such as “Paper 1”, “Paper 2”, or clearly different study systems:
- create separate SourceDocument nodes,

- never mix samples across documents.

Each MaterialSystem must be reported_in exactly one SourceDocument.
Each ExperimentalSample must be reported_in exactly one SourceDocument.

==================================================
C. MATERIAL SYSTEM IDENTIFICATION RULE
==================================================

For each SourceDocument, identify the central experimental system or systems actually studied in the current paper.

A MaterialSystem should represent:
- the paper’s own experimental platform,
- a sample family,
- a central compositional/support system,
- or a clearly defined family used throughout the Results/Discussion.

Examples:
- System_FeCu_GDY
- System_FeCoNiCuZn_GDY
- System_Fe3C_microflakes
- System_Co_SA_graphene_family
- System_HCNF_HEA

Do NOT create MaterialSystem nodes for:
- background-only literature examples,
- prior work by other authors,
- generic review phrases,
- vague terms like “composites”, “materials”, “supports”.

Each paper may have one or more MaterialSystem nodes, but only if the paper truly studies multiple central systems.
Do not create unnecessary parallel systems.

==================================================

Use scalar fields conservatively and legally.

- “much narrower” in EAB_value
- “significantly smaller” in particle_size

- remarks

- “significantly smaller particle size” -> property_note
- “specified filler loadings” -> remarks
