You are an expert in scientific knowledge graph analysis for chemistry, materials science, and electrochemistry.

Your task is to **name and summarize communities** in a bottom-level entity graph.

Each community is a set of entities (nodes) with strong connections in the knowledge graph.
For each community, you will receive:
- `id`: a numeric community id
- `size`: number of entities in the community
- `members`: a list of representative entity names (may be truncated)
- `top_relations`: the most frequent relations inside this community, as `(relation, count)` pairs

---

### INPUT

You are given a JSON array named `communities` encoded as text:

```json
{communities_json}
```

Each element has the shape:

```json
{
  "id": 0,
  "size": 12,
  "members": ["entity_1", "entity_2", "..."],
  "top_relations": [["improves", 5], ["inhibits", 3]]
}
```

---

### YOUR GOAL

For **each** community:

1. **Name** the community (`name`):
   - Short, specific, and informative (3–10 English words if possible)
   - Reflect the common topic, entities, or mechanism (e.g., "In(NO3)3 Additives for SEI Stabilization")
   - Avoid special characters other than spaces, hyphens, and parentheses.

2. **Summarize** the community (`summary`):
   - 2–4 sentences.
   - Mention:
     - what the main topic or function is,
     - which types of entities are involved (materials, additives, SEI, performance metrics, etc.),
     - how they are related (e.g., improves cycling stability, suppresses dendrites, forms robust SEI).
   - Use concise scientific language.

---

### OUTPUT FORMAT (VERY IMPORTANT)

Return **only** a JSON array (no explanation text), where each element has:

```json
[
  {
    "id": 0,
    "name": "string",
    "summary": "string"
  }
]
```

Rules:
- The `id` must exactly match the input `id`.
- Do **not** include any extra fields.
- The output must be valid JSON and parseable.
