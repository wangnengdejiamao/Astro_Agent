1. CORE EXTRACTION RULE
==================================================

1. Follow the schema strictly.

3. Always create ExperimentalSample nodes for:

   - comparison / control samples

4. Use:

7. For a single paper, prefer fuller grounded extraction over over-compression, as long as type correctness and field correctness are preserved.

==================================================

ExperimentalSample:

- chemical species

CarbonSupport:

Important examples:

Do not let MaterialSystem absorb sample-level measured scalar facts when ExperimentalSample nodes can be formed.

==================================================
9. FIELD LEGALITY RULE
==================================================

Use scalar fields conservatively and legally.

Allowed:
- RL_min_value -> one reflection loss value in dB
- RL_min_frequency -> one frequency point in GHz
- EAB_value -> one bandwidth width in GHz
- EAB_frequency_range -> one bandwidth interval
- matching_thickness -> one absorber thickness
- particle_size -> one explicit size
- filler_loading -> wt% loading
- precursor_loading_ratio -> a true ratio
- precursor_amount_note -> raw amounts such as “1.5 mmol CuCl, 500 mg TEB”

Do NOT force descriptive phrases into numeric fields.

Examples of illegal mapping:
- “98% absorption” -> not RL_min_value
- “much narrower EAB” -> not EAB_value
- “significantly smaller particle size” -> not particle_size
- “500 mg TEB” -> not filler_loading
- “outer diameter / inner diameter” -> not matching_thickness

If the statement is descriptive rather than a legal scalar value, place it in:
- performance_note
- property_note
- remarks
