CORE EXTRACTION STRATEGY
--------------------------------

Use a dual-granularity strategy:

1. Core sample layer:
- Extract ExperimentalSample nodes for explicit condition-specific samples and measured samples.
- Put explicit scalar performance values on ExperimentalSample whenever clearly sample-specific.

2. Shared knowledge layer:

- Shared non-scalar findings may be attached to MaterialSystem or a family-level ExperimentalSample if they describe the whole sample family rather than one specific condition.
- This shared layer is encouraged in order to preserve more useful nodes.

Do not force every meaningful sentence into sample-specific scalar facts.
If a conclusion is clearly shared across a sample family, keep it as a shared finding rather than dropping it.

ENTITY CREATION RULES
--------------------------------

Create entities only when they are explicit and scientifically useful.

1. SourceDocument
Use for each paper/study.

2. MaterialSystem
Use for the general material platform or sample family.
Examples:
- System_Fe3C_microflakes
- System_FeCu_GDY
- System_Co_SA_graphene

3. ExperimentalSample
Use for:
- explicit sample codes,
- explicit condition-specific samples,
- control samples,
- representative samples with explicit measured performance.

4. MaterialComponent / CarbonSupport
Use for explicit components and supports.

5. SynthesisMethod
Use for named synthesis routes.

6. ProcessingCondition
Use for reusable process concepts.
Examples:
- electrochemical dealloying

7. StructureConcept

SAMPLE-SPECIFIC VS SHARED FACT RULE
--------------------------------

Use this decision rule:

A. If the text gives:
- an explicit sample code or explicit condition,
and
- a clearly corresponding numeric value,
store it on the matching ExperimentalSample.

B. If the text gives:
- a shared conclusion across the whole family,
and
- it is non-numeric or broadly descriptive,
store it as:
- StructureConcept
- CharacterizationFinding
- EMProperty
- LossMechanism
linked to the family-level MaterialSystem or family-level ExperimentalSample.

Examples of shared findings suitable for family/system level:
- irregular flake structure
- orthorhombic crystal structure
- conductive network tendency
- shape anisotropy
- magnetic-loss-dominated behavior
- strip domain structures observed in the family discussion

This rule is intended to preserve more useful nodes while keeping numeric sample facts clean.

--------------------------------
FIELD INTERPRETATION RULE
--------------------------------

Interpret fields conservatively:

- RL_min_value = reflection loss value in dB
- RL_min_frequency = single frequency point for RL_min
- EAB_value = bandwidth width in GHz
- EAB_frequency_range = GHz interval
- matching_thickness = absorber thickness in mm
- epsilon_real / epsilon_imag = complex permittivity quantities
- mu_real / mu_imag = complex permeability quantities

Do not force obviously mismatched values into these fields.
When a sentence is descriptive rather than a clean numeric property statement, prefer:
- CharacterizationFinding
- StructureConcept
- LossMechanism
- EMProperty
- remarks

rather than coercing it into a numeric attribute.
