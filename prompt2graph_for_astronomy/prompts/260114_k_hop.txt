### CORE PRINCIPLES (STRICTLY ENFORCED)

#### 1. ENTITY CANONICALIZATION BASED ON SCHEMA TYPE

##### A. Component-Level Entities (`ChemicalCompound`, `Salt`, `Solvent`, `Additive`)
- Extract **only the canonical chemical name or formula**, stripping:
  - Concentrations ("0.2 M", "5 wt%")
  - Ratios ("3:7"), physical states ("anhydrous" → keep as attribute), procedures ("added", "dissolved")
- Normalize formulas: `LiPF₆` → `LiPF6`, `$In(NO_{3})_{3}·6H_{2}O$` → `In(NO3)3·6H2O`
- Use standard abbreviations only if unambiguous (e.g., "FEC" for fluoroethylene carbonate)

##### B. System-Level Entities (`Electrolyte`, `Anode`, `Cathode`)
- Preserve **essential compositional descriptors**:  
  ✅ "electrolyte with FEC", "NMC811 cathode", "Si anode with carbon coating"  
  ❌ Remove performance claims: "showing high stability"

##### C. Mechanism & State Entities (`InterfacialLayer`, `SideReaction`, `FailureMode`, `MaterialState`, `Phenomenon`)
- Use **concise, standardized names**:
  - ✅ `"SEI"`, `"lithium dendrite growth"`, `"solvent reduction"`, `"delithiated NMC811"`
  - ❌ Avoid full sentences: not `"formation of a stable SEI"`

##### D. `Property` / `PerformanceMetric`
- Value-free names only:  
  ✅ `"cycle stability"`, `"ionic conductivity"`  
  ❌ `"conductivity = 8 mS/cm"`

---

#### 2. CAPTURE MECHANISM CHAINS FOR MULTI-HOP REASONING

When the text describes a **causal or mechanistic chain**, extract **each step as a separate triple** using precise relations:

- If X **forms** Y → use `["X", "forms", "Y"]`  
  (e.g., `["FEC", "forms", "LiF-rich SEI"]`)
- If X **suppresses/inhibits** Y → use `["X", "suppresses", "Y"]`  
  (e.g., `["stable SEI", "suppresses", "lithium dendrite growth"]`)
- If X **leads_to/causes/results_in** Y → use appropriate relation  
  (e.g., `["salt decomposition", "leads_to", "HF generation"]`)
- If a mechanism **explains a performance outcome**, use:  
  `["<mechanism description>", "is_mechanism_for", "<performance metric>"]`  
  (e.g., `["suppression of electrolyte reduction", "is_mechanism_for", "improved cycle stability"]`)

> 🔍 **Never collapse a multi-step mechanism into a single direct link** unless the source explicitly states it (e.g., “FEC directly improves cycle life”).

---

#### 3. TRIPLE DIRECTION MUST RESPECT GRAMMATICAL LOGIC

- Passive voice: **"X was measured by Y"** → `[X, "is_measured_by", Y]`  
- Active voice: follow subject–verb–object order  
- For relations like `suppresses`, `promotes`, `forms`:  
  - Subject = agent/initiator  
  - Object = target/outcome

---

#### 4. SOURCE = CONTIGUOUS, VERBATIM EXCERPT

- 1–2 contiguous sentences that **explicitly support the triple or attribute**
- No paraphrasing, summarization, or combining non-adjacent clauses

---

#### 5. EVIDENCE = EXPLICIT, STRUCTURED REASONING

In `evidence`, explain:
- Whether the sentence is **active/passive**
- Why the **entity was canonicalized** (e.g., “removed concentration per Salt rule”)
- Why the **relation and direction are correct**
- If part of a **mechanism chain**, state:  
  > “This is step [N] in a causal chain: [brief summary].”
- Why omitting/modifying would break multi-hop connectivity

Example:
> Source: "FEC decomposes to form a LiF-rich SEI, which suppresses lithium dendrite growth."
> Evidence: "Two-step mechanism. First: 'FEC' (Additive) forms 'LiF-rich SEI' (InterfacialLayer). Second: 'LiF-rich SEI' suppresses 'lithium dendrite growth' (FailureMode). Both triples are extracted to enable multi-hop reasoning from additive to failure mitigation."
